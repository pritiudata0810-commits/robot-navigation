❓ COMMON QUESTIONS

Q: What does "A*" mean?
A: A* is an efficient pathfinding algorithm that guarantees the shortest path.
   It's used in real robotics and AI systems.

Q: Why a random grid?
A: Every run generates a different grid, so every challenge is unique!
   The AI must solve a different problem each time.

Q: What's the difference between difficulties?
A: More walls = more obstacles = AI needs smarter pathfinding.
   SLOW: 10% walls | MEDIUM: 25% walls | HARD: 40% walls

Q: Can I change the grid size?
A: The grid is fixed at 12×12 for consistency, but the code is modular
   so you could change it (see GridConfig in the source).

Q: Is the path always optimal?
A: Yes! A* algorithm guarantees the shortest path every single time.

Q: How long does it take?
A: Path calculation: < 500ms | Animation: varies by speed selected

Q: Can I run it multiple times?
A: Yes! Every run creates a completely new random grid.


🔧 TROUBLESHOOTING

ISSUE: "ModuleNotFoundError: No module named 'rich'"
FIX: pip install rich

ISSUE: Grid looks too small or cramped
FIX: Resize your terminal window to be wider (120+ characters)

ISSUE: Colors look wrong
FIX: Ensure your terminal supports 24-bit colors

ISSUE: Simulation runs slowly
FIX: Try "FAST" speed instead of "SLOW"

ISSUE: Path calculation takes long
FIX: This is normal for HARD difficulty. Wait about 1 second.

ISSUE: Can't see the full dashboard
FIX: Make your terminal wider and taller
