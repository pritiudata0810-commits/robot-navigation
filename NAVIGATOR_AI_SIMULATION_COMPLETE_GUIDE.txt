HOW THE SIMULATION WORKS
================================================================================

PHASE 1: GRID GENERATION
  Input: Difficulty level selected by user
  Process:
    • Creates empty 20×20 grid
    • Randomly places obstacles based on difficulty percentage
    • Places robot at top-left (0, 0)
    • Places goal at bottom-right (19, 19)
    • Validates goal is reachable
    • If unreachable, regenerates until goal is reachable
  Output: Random grid ready for simulation

PHASE 2: PATHFINDING SEARCH
  Input: Grid with robot, obstacles, and goal
  Algorithm: A* (A-Star)
  Process:
    • Maintains open set (nodes to evaluate)
    • Maintains closed set (nodes already evaluated)
    • Calculates scores:
      - g_score = cost from start node
      - h_score = estimated cost to goal (Manhattan distance)
      - f_score = g_score + h_score
    • Explores lowest f_score nodes first
    • Expands neighbors and updates scores
    • Records each explored node (shown as cyan)
    • Terminates when goal is found
  Output: Optimal path from robot to goal

PHASE 3: PATH VISUALIZATION
  Input: Optimal path computed by A*
  Process:
    • Highlights entire path in yellow
    • Shows all explored nodes in cyan
    • Prepares robot for animation
  Output: Visual representation of solution

PHASE 4: ROBOT ANIMATION
  Input: Speed setting, optimal path
  Process:
    • Moves robot one cell at a time
    • Delay between moves based on speed:
      - Slow = 500ms per move
      - Normal = 200ms per move
      - Fast = 50ms per move
    • Updates telemetry in real-time:
      - Robot position
      - Steps taken
      - Elapsed time
    • Shows bright cyan dot representing robot
  Output: Smooth animation of robot movement

PHASE 5: COMPLETION
  Trigger: Robot reaches goal
  Process:
    • Stops animation
    • Updates status to "GOAL REACHED"
    • Shows completion message box with statistics
    • Displays:
      - Total Steps: How many cells the robot moved
      - Elapsed Time: Total simulation duration
      - Path Length: Number of cells in optimal path
      - Nodes Explored: How many cells A* evaluated
  Output: Victory message and replay options

================================================================================
A* ALGORITHM EXPLANATION
================================================================================

What is A*?
  • Optimal pathfinding algorithm
  • Guarantees shortest path (for uniform movement costs)
  • More efficient than brute-force search
  • Combines Dijkstra's algorithm with heuristic guidance

How A* Works in This Simulator:
  1. Start at robot position (0, 0)
  2. For each node, calculate:
     - g_score: Distance traveled from start
     - h_score: Estimated distance to goal (Manhattan distance)
     - f_score: g_score + h_score (total estimated cost)
  3. Always explore the node with lowest f_score first
  4. Expand neighbors and update their scores
  5. Track path backwards from goal to start
  6. Return reversed path (start to goal)

Manhattan Distance Heuristic:
  • Distance = |x1 - x2| + |y1 - y2|
  • Works for 4-directional movement (no diagonals)
  • Examples:
    - (0,0) to (3,4): distance = 3 + 4 = 7
    - (5,5) to (5,8): distance = 0 + 3 = 3

Why A* is Good:
  ✓ Finds optimal shortest path
  ✓ Explores fewer nodes than exhaustive search
  ✓ Heuristic guides exploration toward goal
  ✓ Efficient for large grids

================================================================================
TROUBLESHOOTING
================================================================================

"customtkinter not found" error
  Solution: pip install customtkinter
  
"No module named tkinter" error
  Solution: tkinter included with Python on most systems
           On Linux: sudo apt-get install python3-tk
  
"Grid looks wrong" error
  Solution: Ensure window size is at least 1400×700
  
"Robot doesn't move" error
  Solution: Wait for pathfinding to complete (appears frozen during search)
           Check if speed is set to Slow (500ms per move)
  
"Goal unreachable" loop
  Solution: Rare scenario - grid is regenerated automatically
           If persists, try generating new grid manually
