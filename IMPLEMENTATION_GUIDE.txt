================================================================================
TECHNICAL DETAILS FOR DEVELOPERS
================================================================================

WHY UNIVERSAL [/] WORKS
  Rich library recognizes [/] as universal closing tag:
  - [bold]text[/] ✓ VALID
  - [#RRGGBB]text[/] ✓ VALID
  - [bold #RRGGBB]text[/] ✓ VALID
  
  But specific closings don't work:
  - [/bold] ✗ Only works with [bold]
  - [/#RRGGBB] ✗ Color closing doesn't exist
  - [/color] ✗ Not a real closing tag

WHAT CAUSED THE CRASH
  Original code used [/#00FFFF] as closing tag, but [#00FFFF] is opening.
  Rich parser expected [/] or specific matching tag, threw MarkupError.
  
  Example line that crashed:
    f"[#00FFFF]text[/#00FFFF]"  ← WRONG
  
  Should be:
    f"[#00FFFF]text[/]"  ← CORRECT

ERROR HANDLER IMPROVEMENT
  Old: self.console.print(f"[red]❌ Error: {e}[/red]")
  
  Problem: If {e} contains special characters or brackets, it breaks markup.
  
  New: self.console.print(f"[bold red]❌ SYSTEM ERROR: [/]{str(e)}")
  
  Solution: Separates prefix (safe, marked up) from message (plain text).
