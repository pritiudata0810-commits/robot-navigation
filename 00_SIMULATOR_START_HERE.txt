🔧 TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

PROBLEM: "No module named 'PySide6'"
SOLUTION: 
  pip install pyside6
  Then retry: python robot_navigation_simulator.py

PROBLEM: "No module named 'qt_material'"
SOLUTION:
  pip install qt-material
  OR: Application will work without it (just less styled)

PROBLEM: Application window doesn't appear
SOLUTION:
  1. Check Python version: python --version (need 3.7+)
  2. Verify PySide6 works:
     python -c "from PySide6.QtWidgets import QApplication; print('OK')"
  3. Try explicit startup:
     python -u robot_navigation_simulator.py

PROBLEM: GUI freezes during simulation
SOLUTION: This shouldn't happen! If it does:
  • Check system resources
  • Try with fewer obstacles (Easy difficulty)
  • Update PySide6: pip install --upgrade pyside6
  • Restart computer

PROBLEM: Graphics rendering looks poor
SOLUTION:
  • Update display drivers
  • Check display resolution (set to native)
  • Disable scaling in Windows display settings
  • Try different Qt rendering: set QT_QPA_PLATFORM=offscreen
