================================================================================
A* ALGORITHM DETAILS
================================================================================

Heuristic Function: Manhattan Distance
  Formula: |x1 - x2| + |y1 - y2|
  Why: Optimal for 4-directional movement

Pathfinding Scores:
  • g_score: Actual cost from start node
  • h_score: Estimated cost to goal (heuristic)
  • f_score: Total cost (g_score + h_score)

Search Strategy:
  • Always explore lowest f_score node first
  • Uses open set (to evaluate) and closed set (evaluated)
  • Expands neighbors and updates scores
  • Maintains path reconstruction data
  • Guarantees optimal shortest path

Movement: 4-directional (up, down, left, right)
  No diagonals to keep pathfinding clean and understandable
