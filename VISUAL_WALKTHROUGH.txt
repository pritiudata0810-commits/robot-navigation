════════════════════════════════════════════════════════════════════════════════
KEY METRICS EXPLAINED
════════════════════════════════════════════════════════════════════════════════

TOTAL STEPS
  The number of moves the robot takes from start to goal
  Typically: 25-40 steps on a 20×20 grid
  Example: Robot takes 23 steps to reach goal

PATH LENGTH
  The length of the optimal path found by A*
  Usually 3-5 steps shorter than actual movement
  Guaranteed to be shortest by A* algorithm
  Example: Optimal path is 19 cells long

NODES EXPLORED
  How many cells A* algorithm evaluated
  Shows search efficiency
  Lower = More efficient algorithm
  Example: Algorithm explored 47 cells to find path

ELAPSED TIME
  Actual wall-clock time for simulation
  Depends on speed setting (Slow/Normal/Fast)
  Shows real performance timing
  Example: 3.25 seconds total runtime

EFFICIENCY %
  Path Length / Nodes Explored × 100
  Higher = More efficient search
  Perfect = 100% (unrealistic)
  Good = 30-50%
  Example: 19/47 × 100 = 40.4% efficiency
