OBJECT-ORIENTED STRUCTURE

MainWindow (QMainWindow)
├── Manages application window and screen navigation
├── Connects signals between screens
└── Handles theme application

WelcomeScreen (QWidget)
├── Welcome title and subtitle
├── Start, About, Exit buttons
└── Signals to transition to configuration

ConfigurationScreen (QWidget)
├── Difficulty level selection (radio buttons)
├── Simulation speed selection (radio buttons)
└── Signals to start simulation with selected options

SimulationDashboard (QWidget)
├── Movement history log (left panel)
├── GridCanvasWidget - visualization (center)
└── Telemetry display (right panel)
└── Control buttons (bottom)

AnalyticsSummaryScreen (QWidget)
├── Metric cards showing results
└── Back button to return to configuration

GridCanvasWidget (QGraphicsView)
├── Renders grid, obstacles, robot, goal
├── Visualization of explored nodes
└── Path highlighting

SimulationEngine (QThread)
├── Runs pathfinding in background thread
├── Emits signals for UI updates
├── Manages robot movement
└── Thread-safe pause/resume/stop

AStarPathfinder
├── Core A* algorithm implementation
├── Node exploration tracking
└── Path reconstruction

GridEnvironment
├── 20×20 grid management
├── Obstacle generation
├── Position validation

RobotAgent
├── Robot state and position
├── Path following
└── Movement history recording

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

ISSUE: "ModuleNotFoundError: No module named 'PySide6'"
SOLUTION: pip install --upgrade pyside6

ISSUE: Application window doesn't appear
SOLUTION: 
  1. Check Python version: python --version (need 3.7+)
  2. Verify PySide6: python -c "from PySide6.QtWidgets import QApplication"
  3. Try explicit run: python -u robot_navigation_simulator.py

ISSUE: "No module named 'qt_material'"
SOLUTION: pip install qt-material
NOTE: Application works without it (just less styled)

ISSUE: GUI freezes during simulation
SOLUTION: This shouldn't happen due to threading. If it does:
  1. Check for blocking operations in main thread
  2. Verify signals are connected correctly
  3. Ensure QThread.start() is called, not run()
